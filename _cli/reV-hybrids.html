reV-hybrids

reV Hybridization Command Line Interface

reV-hybrids [OPTIONS] COMMAND [ARGS]...

Options

--version

Show the version and exit.

-n, --name <name>

reV Hybrids job name.

Default

docs

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

direct

reV hybridization CLI.

reV-hybrids direct [OPTIONS] COMMAND [ARGS]...

Options

-s, --solar_fpath <solar_fpath>

Required Filepath to solar rep profile file.

-w, --wind_fpath <wind_fpath>

Required Filepath to wind rep profile file.

-so, --allow_solar_only

Flag to to allow SC points with only solar capcity (no wind) in hybrid profiles.

-wo, --allow_wind_only

Flag to to allow SC points with only wind capcity (no solard) in hybrid profiles.

-fna, --fillna <fillna>

String representation of a dictionary of fill values for merged columns {col_name: val} where col_name is the name of a merged column (prefixed by solar_ or wind_) and val is the values used to fill any n.a. values resulting from a merge (e.g. wind column values for a row consisting purely of solar capacity).

-l, --limits <limits>

String representation of a dictionary specifying a mapping of {colum_name: max_value} representing the upper limit (maximum value) for the values of a column in the merged meta. For example, limits {solar_capacity: 100} would limit all the values of the solar capacity in the merged meta to a maximum value of 100. This limit is applied BEFORE ratio calculations. The names of the columns should match the column names in the merged meta, so they are likely prefixed by solar_ or wind_. By default, None (no limits applied).

-rb, --ratio_bounds <ratio_bounds>

List of two floats representing the lower and upper bounds on the ratio of the two columns specified with the ratio input, which is set to solar_capacity/wind_capacity by default. The same value can be used as the upper and lower bound to represent a single ratio. For example, ratio_bounds=[0.5, 1.5] would adjust the values of both of the ratio columns such that their ratio is always between half and double (e.g., no value would be more than double the other). To specify a single ratio value, use the same value as the upper and lower bound. For example, ratio_bounds=(1, 1) would adjust the values of both of the ratio columns such that their ratio is always equal.

-r, --ratio <ratio>

String representation of the ratio calculation used to calculate the ratio that is limited by the ratio_bounds`input. This string must be in the form numerator_column_name/denominator_column_name. For example, `ratio solar_capacity/wind_capacity would limit the ratio of the solar to wind capacities as specified by the ratio_bounds input. If ratio_bounds is not specified, this input does nothing. The names of the columns should match the column names in the merged meta, so they are likely prefixed by solar_ or wind_.

Default

solar_capacity/wind_capacity

-od, --out_dir <out_dir>

Directory to save rep profile output h5.

Default

./

-ld, --log_dir <log_dir>

Directory to save rep profile logs.

Default

./logs/

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

slurm

slurm (Eagle) submission tool for reV representative profiles.

reV-hybrids direct slurm [OPTIONS]

Options

-a, --alloc <alloc>

Required SLURM allocation account name.

-mem, --memory <memory>

SLURM node memory request in GB. Default is None

-wt, --walltime <walltime>

SLURM walltime request in hours for single year rep_profiles run. Default is 1.0

Default

1.0

-l, --feature <feature>

Additional flags for SLURM job. Format is qos=high or depend=[state:job_id]. Default is None.

-env, --conda_env <conda_env>

Conda env to activate

-mod, --module <module>

Module to load

-sout, --stdout_path <stdout_path>

Subprocess standard output path. Default is in out_dir.

-sh, --sh_script <sh_script>

Extra shell script commands to run before the reV call.

from-config

Run reV hybridization from a config file.

reV-hybrids from-config [OPTIONS]

Options

-c, --config_file <config_file>

Required reV hybrid profiles configuration json file.

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

valid-config-keys

Echo the valid Hybrids config keys

reV-hybrids valid-config-keys [OPTIONS]
