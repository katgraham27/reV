reV-nrwal

reV-NRWAL Command Line Interface

reV-nrwal [OPTIONS] COMMAND [ARGS]...

Options

--version

Show the version and exit.

-n, --name <name>

reV NRWAL job name.

Default

docs

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

direct

Main entry point to run reV-NRWAL analysis

reV-nrwal direct [OPTIONS] COMMAND [ARGS]...

Options

-gf, --gen_fpath <gen_fpath>

Required reV wind generation/econ output file. Anything in the output_request is added and/or manipulated in this file.

-sd, --site_data <site_data>

Required Site-specific input data for NRWAL calculation. String should be a filepath that points to a csv. Rows match sites, columns are input keys. Need a “gid” column that corresponds to the “meta_gid_col” in the gen_fpath meta data and a “config” column that corresponds to the nrwal_configs input. Only sites with a gid in this files “gid” column will be run through NRWAL.

-sf, --sam_files <sam_files>

Required SAM config files lookup mapping config keys to config filepaths. (required) (dict). Should have the same config keys as the nrwal_configs input.

-nc, --nrwal_configs <nrwal_configs>

Required NRWAL config files lookup mapping config keys to config filepaths. (required) (dict). Should have the same config keys as the sam_files input.

-or, --output_request <output_request>

List of output dataset names you want written to the gen_fpath file. Any key from the NRWAL configs or any of the inputs (site_data or sam_files) is available to be exported as an output dataset. If you want to manipulate a dset like cf_mean from gen_fpath and include it in the output_request, you should set save_raw=True and then in the NRWAL equations use cf_mean_raw as the input and then define cf_mean as the manipulated data that will be included in the output_request.

-sr, --save_raw <save_raw>

Flag to save a copy of existing datasets in gen_fpath that are part of the output_request. For example, if you request cf_mean in output_request and manipulate the cf_mean dataset in the NRWAL equations, the original cf_mean will be archived under the “cf_mean_raw” dataset in gen_fpath.

-mg, --meta_gid_col <meta_gid_col>

Column label in the source meta data from gen_fpath that contains the unique gid identifier. This will be joined to the site_data “gid” column.

-mc, --site_meta_cols <site_meta_cols>

Column labels from site_data to pass through to the output meta data. None (default) will use class variable DEFAULT_META_COLS, and any additional cols requested here will be added to DEFAULT_META_COLS.

-ld, --log_dir <log_dir>

Directory to save reV-NRWAL logs.

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

slurm

slurm (Eagle) submission tool for reV supply curve aggregation.

reV-nrwal direct slurm [OPTIONS]

Options

-a, --alloc <alloc>

Required SLURM allocation account name.

-l, --feature <feature>

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-mem, --memory <memory>

SLURM node memory request in GB. Default is None

-wt, --walltime <walltime>

SLURM walltime request in hours. Default is 1.0

-mod, --module <module>

Module to load

-env, --conda_env <conda_env>

Conda env to activate

-sout, --stdout_path <stdout_path>

Subprocess standard output path. Default is in out_dir.

-sh, --sh_script <sh_script>

Extra shell script commands to run before the reV call.

from-config

Run reV-NRWAL analysis from a config file.

reV-nrwal from-config [OPTIONS]

Options

-c, --config_file <config_file>

Required reV-NRWAL configuration json file.

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

valid-config-keys

Echo the valid nrwal config keys

reV-nrwal valid-config-keys [OPTIONS]
